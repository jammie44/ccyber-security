# CyberPlatform

AI-powered cybersecurity platform — production-ready implementation scaffold.

This repository contains the working source code implementation derived from the
full architecture specification (10 modules + Final Deliverable system architecture).
It is structured for deployment directly on **Render.com**, with no Docker,
Docker Compose, or Kubernetes manifests involved.

## Stack

- **Backend:** Python 3.12 / FastAPI / SQLAlchemy 2.0 (async) / Alembic / Celery / Redis
- **Frontend:** React 18 / TypeScript / Vite / TailwindCSS / TanStack Query / Zustand / Chart.js
- **Database:** PostgreSQL
- **AI:** Anthropic Claude API (risk explanation, Module 9)

## Local development

### Backend

```bash
cd backend
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # edit DATABASE_URL / REDIS_URL for your local setup
alembic upgrade head
uvicorn app.main:app --reload
```

API docs available at `http://localhost:8000/api/v1/docs`.

### Frontend

```bash
cd frontend
npm install
cp .env.example .env
npm run dev
```

### Celery worker + beat (optional locally)

```bash
celery -A app.tasks.celery_app worker --loglevel=info -Q default,scoring,monitoring,discovery
celery -A app.tasks.celery_app beat --loglevel=info
```

## Running tests

```bash
# Backend unit tests (no database required)
python3 -m pytest tests/backend/unit/ -v

# Backend API integration tests (requires a live PostgreSQL test DB at DATABASE_URL;
# skips automatically if unreachable)
python3 -m pytest tests/backend/api/ -v

# Frontend component tests
cd frontend && npm test
```

## Deploying to Render

1. Push this repository to GitHub/GitLab.
2. In the Render dashboard, choose **New > Blueprint** and point it at this repo —
   Render will detect `render.yaml` at the repo root and provision all services:
   - `cyberplatform-api` (FastAPI web service, runs `alembic upgrade head` on every deploy)
   - `cyberplatform-worker` (Celery worker)
   - `cyberplatform-beat` (Celery beat scheduler)
   - `cyberplatform-web` (Vite-built frontend, served via `vite preview`)
   - `cyberplatform-redis` (managed Redis)
   - `cyberplatform-db` (managed PostgreSQL)
3. Set the `sync: false` environment variables in the Render dashboard for `cyberplatform-api`
   and `cyberplatform-worker`:
   - `ANTHROPIC_API_KEY` — required for Module 9 AI risk explanation endpoint
   - `ELASTICSEARCH_URL` — optional, leave blank to disable search features
   - `KAFKA_BOOTSTRAP_SERVERS` — optional, leave blank to disable event streaming
4. Deploy. `SECRET_KEY` is auto-generated by Render (`generateValue: true`); database and Redis
   connection strings are wired automatically via `fromDatabase` / `fromService`.
5. Once live, the frontend's `VITE_API_BASE_URL` in `render.yaml` should match your actual
   `cyberplatform-api` service URL (update if your Render service name differs from the default).

## Notes on scope

This implementation covers Modules 1 (Asset Inventory), 2 (Auth/Identity), 4 (Vulnerability
Management), 5 (Risk Scoring Engine), and a working slice of 8 (Continuous Monitoring — alert
firing/dedup/acknowledgement) and 9 (AI Analysis Layer — risk score explanation endpoint), reflecting
the priority order specified in the implementation phase prompt. Modules 3, 6, 7, and 10 are fully
specified in the architecture documentation set but not yet implemented as running code in this
scaffold — they follow the same repository/service/schema/endpoint pattern established here and are
the natural next extension point.
